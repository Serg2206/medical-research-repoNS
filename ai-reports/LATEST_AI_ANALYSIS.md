## 📁 scripts/fix_csv_headers.py

**Строк кода:** 29

### 🎯 AI Рекомендации:

В целом, код написан хорошо и легко читается. Однако, есть несколько моментов, которые можно улучшить:

1. **Производительность**: Вместо того, чтобы сначала считывать все строки из файла, а затем их обрабатывать, можно считывать и обрабатывать строки по одной. Это позволит снизить потребление памяти, особенно для больших файлов.

2. **Безопасность**: Скрипт перезаписывает исходный файл, что может быть опасно. Если в процессе работы скрипта произойдет ошибка, исходный файл может быть поврежден. Лучше записывать результат в новый файл, а затем, если все прошло успешно, заменить исходный файл на новый.

3. **Читаемость и best practices**: Вместо того, чтобы использовать индекс `i` для определения первой строки, можно использовать флаг, который будет установлен в `True` после обработки первой строки. Это улучшит читаемость кода.

Вот пример улучшенного кода:

```python
#!/usr/bin/env python3
"""Fix CSV file by removing invalid header rows"""

import csv
import os
import sys

input_file = "training-data/medical_training_data.csv"
output_file = "training-data/medical_training_data_cleaned.csv"

# Counter for valid and invalid lines
valid_lines = 0
invalid_lines = 0

# Open the input CSV file and a temporary output file
with open(input_file, "r", encoding="utf-8") as fin, open(output_file, "w", encoding="utf-8") as fout:
    # Process each line
    for i, line in enumerate(fin):
        # Keep the header row or lines with CSV data
        if i == 0 or "," in line:
            fout.write(line)
            valid_lines += 1
        else:
            print(f"Removing invalid line {i+1}: {line.strip()}")
            invalid_lines += 1

# Replace the original file with the cleaned file
os.replace(output_file, input_file)

print(
    f"\nDone! Kept {valid_lines} valid lines, removed {invalid_lines} invalid lines"
)
```

Этот код делает то же самое, что и исходный, но он более безопасен и эффективен.
